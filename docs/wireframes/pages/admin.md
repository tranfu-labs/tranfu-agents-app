## 注释

| 编号 | 元素 | 状态/交互 | 数据来源 | 引用控件 |
|---|---|---|---|---|
| ① | 钥匙浮层 | 进页即弹；输入存 sessionStorage，关页失效；之后请求带 `X-TF-Admin-Key`；403→原地报错+denied 审计 | 用户输入 | 弹窗 |
| ② | 警示条 | 常驻，提示不可恢复 | 固定文案 | 卡片 |
| ③ | 实体标签页 | 切 Operators/Identities/Sessions/Skills + 回收站 Trash；切 tab 即换 ⑤ 列口径 | 客户端状态 | 标签页 |
| ④ | 工具条 | 名称搜索（服务端）+ 运行时筛选 + 仅活跃 + 刷新 | 客户端/查询参数 | 工具栏 |
| ④b | 日期清理 | 输入 operator 与 before_day（UTC 日期），触发同一个 preview/delete 流程；禁止无 operator 的全局日期清理 | `targets[].before_day` | 表单 |
| ⑤ | 清单表格 | 整行勾选；取数 `events∪skill_uses`（含裁剪后的 skill_uses 孤儿、空 operator 垃圾行）；活跃中标 ●；Skills tab 列改为 skill·used·equipped·首见 | `GET /api/admin/inventory` | 表格 |
| ⑥ | 操作条 | 显示已选数与是否含活跃；触发 ⑦ | 客户端选择 | 工具栏 |
| ⑦ | 预览面板 | dry-run 逐表行数 + active session 明细 + 副作用（受影响 operator、first_day 漂移、清掉的卡）+ preview_token | `POST /api/admin/preview` | 卡片 |
| ⑧ | 二次确认浮层 | 超 `TF_ADMIN_MAX_ROWS` 或跨 >1 operator 时须手输行数；删除回带 token，集合变→409 | `DELETE /api/admin/data` | 弹窗 |
| ⑨ | 回收站 Trash | 列批次（选择器/行数/状态）；整批恢复，键冲突回报 | `GET /api/admin/trash` / `POST /api/admin/restore` | 表格 |
| ⑩ | 结果 Toast | 回显各表实删行数后自动刷新清单 | 删除响应 | 文字块 |
| ⑪ | 导出 DB 按钮 | 顶栏；点击下载整库一致快照（`VACUUM INTO`），文件名带 UTC 时间戳；导出态禁用并显示 loading，成功/失败走 Toast | `GET /api/admin/export` | 按钮 |
