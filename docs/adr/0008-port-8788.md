# ADR-0008 服务端默认端口 8788

- 状态:Accepted
## 背景
部署环境上 8787 被占用,开发改用 8788;仓库各处需与实际部署一致。
## 决策
服务端默认端口、`Dockerfile`(PORT/EXPOSE/CMD)、Coolify Compose 的内部 `expose` 端口、文档与 Tunnel 示例
统一为 **8788**。Coolify Domain 配置写 `https://域名:8788`(这里的 `:8788` 是容器内部端口);
Cloudflare Tunnel 的 `service` 指 `http://localhost:8788`(整站含 `/api/*`、`/v1/*`)。
## 后果
- ✅ 全链路端口一致,避免 Tunnel/验证指错端口。
- 约束:改端口要同时改 app/Dockerfile/compose/文档四处。
