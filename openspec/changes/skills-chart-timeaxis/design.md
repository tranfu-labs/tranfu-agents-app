# 设计:skills-chart-timeaxis

## 业务线怎么串(改动落点)
```
shim / MCP 上报 ──POST /v1/events──▶ skill_uses 表(一行 = session×skill×mode,day 为 UTC 日)
                                          │  (本变更不动:表结构 / 聚合口径 / 上报链路)
                       ┌──────────────────┴───────────────────┐
            GET /api/skills?days=N                   GET /api/skill/{name}
            └ skills_overview() → daily/table/funnel  └ skill_detail_payload() → daily(used/equipped)
              ＋本变更:响应加 today                     ＋本变更:响应加 today
                                          │
                          frontend/(React SPA,内联 SVG 组件)
            StackedSkillChart「每日使用」主图      DetailTrend 详情趋势图
              ＋本变更:横轴铺满窗口 / 悬停浮窗 / 今日进行中
```
本变更只动**读侧**(board 域):`server/app.py` 两个读接口各加 1 行,前端 `frontend/`
两个图表组件 + 一个新浮窗组件 + 窗口选择器。

## 锁定口径(访谈逐项拍板,实现时不得擅改)
- **铺满只填真实 0**:绝不造数据(不走演示用的 `gen30` 之类伪序列);空白天如实留白。
- **横轴范围**:右端 = 服务端 `today`(UTC);左端 = `today-(N-1)`。主图 N ∈ {7,30,90},
  **去掉「全部」档**;详情趋势图固定 N=30(详情页无窗口选择器)。
- **今日 = 序列最后一格**,视为"进行中"(还在累计),用斜纹 / 半透明 + 描边区分,浮窗注明。
- **为什么后端要给 today**:`skill_uses.day` 是 UTC 日;前端若用浏览器本地 `new Date()` 推算窗口
  右端,跨 UTC 午夜会与数据日期错一格,今日柱可能落到错误槽位或丢失。由服务端给权威 `today`、
  前端只据此回推——这是"准确优先"的必要项,成本约 1 行 / 接口。
- **浮窗合计**口径 = 当天所有 skill 的 sessions 之和(含并入"其它"的部分),与堆叠柱总高一致。
- Top8 + 其它分色、max 取各天总量峰值、轴标签每 N 天一个——**沿用现状**,不改。
- used / equipped 仍**分列不相加**(详情图 used 柱 + equipped 折线),沿袭既有约束。

## 改动文件与职责

### `server/app.py`(读接口,各 ~1 行)
1. `skills_overview()`:响应新增 `today`(UTC 当日,ISO 日期串)。`daily` / `table` / `funnel`
   的聚合与口径**全部不变**。`days` 参数:去掉 `0`(全部),校验集合改为 `{7,30,90}`。
2. `skill_detail_payload()`:响应新增 `today`。`daily`(used / equipped 分列)聚合**不变**。
- 铺满是前端职责,后端不预先补零行(避免把"几十 skill × 90 天"的空行塞进响应)。

### `frontend/`(主改区)
- **窗口选择器**:options 由 `[7,30,90,0]` 改为 `[7,30,90]`,默认仍 30;`skillFilters.days` 不再取 0。
- **`stackedChart()`(主图)**:
  - 横轴来源从「`Object.keys(byDay)`(有数据的天)」改为「以 `today` 为右端、回推 `days-1` 天
    得到的完整日期序列」。
  - 每个日期一个槽位:`byDay` 命中才画堆叠柱,未命中留白(仅基线 / 可选极淡刻度)。
  - 宽度 / 轴标签 / Top8+其它分色 / max 峰值——沿用现有逻辑(`w = days.length*28+50` 等)。
  - 今日(序列最后一格)若有量,柱体加"进行中"样式(斜纹或半透明 + 描边)。
- **明细浮窗(新增组件,主图与详情共用)**:
  - 一个绝对定位浮层,锚定当前柱子(非光标);内容 = 日期 + 当天各 skill 按 sessions 降序(色点 + 名 + 值,
    Top8 外并"其它")+ 分隔线 + 合计;今日额外标注"今日进行中"。
  - 替代现在 `<rect><title>` 的原生提示。
  - 触发:悬停某"列"(整天)→ 该列高亮、其余降透明(新增"按天悬停"状态,与图例 hover 的
    `hoverSkill` 解耦,二者不互相覆盖);移出清除。
  - 移动端无悬停 → 点击某列显示浮窗,点别处关闭。
  - 浮窗定位锚定柱子几何(非光标):默认贴柱子右侧、顶部对齐图表顶(chart.top);碰视口右边界翻到柱子左侧,碰下边界整体上移贴视口底,避免溢出。
- **`detailTrend()`(详情趋势图)**:
  - 横轴改为"以 `today` 为右端、回推 29 天"的完整 30 日序列(详情页固定 30 天)。
  - used 柱 + equipped 折线照旧不堆叠相加;空天留白;今日进行中样式。
  - 浮窗显示当天 used / equipped。

## 实现前必须先验证的两件事
1. `skill_uses.day` 确为 UTC 日(ingest 用 `recv[:10]`,UTC),且与服务端 `today` 同一套写法——
   否则今日柱错位。抽样比对一条记录的 day 与 `datetime.now(timezone.utc).date()`。
2. 现有 `chart-box` 在 90 天 × 柱宽下横向滚动仍可用(`min-width` 复用现有 `w` 公式,不破窄屏)。

## 交互场景全集与边界口径(对照 OpenRouter rankings,逐项定状态)
原始诉求是"各种交互场景去优化",截图锁定的是 #1–#3;其余是参照 OpenRouter 后逐项过一遍的结论
(含**铺满轴衍生出来的新边界** #4 #5,不定会在上线后踩坑)。

| # | 场景 | 状态 | 口径 / 说明 |
|---|---|---|---|
| 1 | 时间轴逐日铺满 | **本轮做** | 右端 = 服务端 `today`,左端 = `today-(N-1)` |
| 2 | 悬停整列 + 明细浮窗 | **本轮做** | 日期 + 当天各 skill 降序 + 合计,替原生 title |
| 3 | 今日"进行中"柱 | **本轮做** | 斜纹 / 半透明 + 描边,浮窗标注 |
| 4 | 整窗全空怎么显示 | **本轮做(默认空态)** | `rows` 为空 → 显示空态文案,不渲染一排空轴;只要 ≥1 天有量就铺满 |
| 5 | 筛选 / 搜索后联动 | **本轮做** | 搜索 / runtime / 来源筛选后,某天归零 = 空槽;全空走 #4;图与主表口径一致 |
| 6 | 线性 / 对数(Log)切换 | 不做 | 访谈明确不要 |
| 7 | 「全部」时间档 | 移除 | 改为仅 7 / 30 / 90 天 |
| 8 | 日 / 周 / 月粒度切换 | 不做 | 固定日粒度(去掉全部档后无超长轴问题) |
| 9 | 图例点击隔离单条 skill | 延后 | 现保留"图例 hover 变暗";点击隔离下一轮再评估 |
| 10 | 「其它」段下钻展开 | 延后 | 浮窗"其它"仅给合计;展开看名单下一轮 |
| 11 | X 轴标签密度自适应 | 沿用 | 现有每 N 天一标签;90 天档需走查不挤、不重叠 |
| 12 | 移动端触摸退化为点击 | **本轮做** | 见浮窗规则(无悬停 → 点击显示 / 再点关闭) |

> 需你点头的默认:#4 整窗全空 → 走**空态文案**(而非一排空槽);#9 图例点击隔离、#10「其它」下钻
> 暂**延后**。不否决即按此执行。
