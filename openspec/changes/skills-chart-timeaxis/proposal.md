# 变更提案:skills-chart-timeaxis(SKILLS 图表时间轴铺满与悬停明细)

- 状态:Proposed
- 关联:skills-stats-page(前身:每日柱状图与单 skill 详情趋势图)、specs/board

## 背景 / 问题
SKILLS 上线后,「每日使用」主图与单 skill 详情趋势图的横轴都是"只把有数据的天排上去"
(后端 `GROUP BY day` 的结果直接当 X 轴)。早期数据稀疏时(例如只有今天有上报),整张图塌成
孤零零一根柱,看不出"这是 30 天里只有 1 天有量"还是"统计坏了",空档也无从比较。

参照 OpenRouter rankings:先把所选时间段的**每一天**都铺在横轴上,哪天有数据就在对应位置
长出堆叠柱,没数据就是空槽——图永远是一条连续时间线;悬停某根柱时,弹出当天各项明细 + 合计。

本变更把"铺满 + 悬停明细"补齐到主图与详情趋势图。定位经访谈确认为**内部分析工具、准确优先**:
铺满只用真实的 0,不造任何数据,空白天如实留白——这比"只显示有数据的天"**更准确**,
避免把单日误读为全貌。

## 目标(经访谈逐项拍板,口径细节见 design.md)
- 主图横轴按**所选时间窗逐日铺满**(7 / 30 / 90 天);窗口选择器**去掉「全部」档**。
- 悬停整列:当前列高亮、其余列降透明;弹出**跟随光标的明细浮窗**(日期 + 当天各 skill 降序明细 +
  合计),替代现在简陋的原生 title 提示。
- **今日柱**用"进行中"视觉(斜纹 / 半透明 + 描边)并在浮窗标注,避免今天"还没攒满"被误读为下跌。
- 详情趋势图套用同一套:固定**最近 30 天**逐日铺满 + 同款浮窗(used 柱 + equipped 折线)+ 今日进行中。
- 后端为前端提供权威的 UTC「今天」,杜绝用浏览器本地时间铺轴时区错一格。

## 非目标
- 不换图表类型(仍是堆叠柱,OpenRouter 同款);不引入图表库。
- **不做**线性 / 对数(Log)切换(访谈中明确不要)。
- 不动采集链路与 `skill_uses` 表结构;不改 daily / table / funnel 的聚合口径——铺满是纯前端职责。
- 不动主表、漏斗,以及详情页除趋势图外的其余区块。

## 方案概述(详见 design.md)
后端两个只读聚合接口各加一个 `today` 字段(UTC 当日),其余原样返回。前端在 `stackedChart` /
`detailTrend` 里把横轴改为"以 `today` 为右端、回推 N 天的完整日期序列",并新增一个跟随光标的
明细浮窗组件与"按天悬停"高亮状态;今日柱加"进行中"样式;窗口选择器去掉「全部」。

## 影响
- specs/board:修改柱状图前端规则(横轴铺满、悬停明细、今日进行中);接口去掉 `days=0`、
  两接口响应各加 `today`。
- 不触碰 ingest 域;表结构、上报链路、聚合口径均不变。
