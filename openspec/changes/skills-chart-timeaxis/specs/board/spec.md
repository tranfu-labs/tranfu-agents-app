# spec delta:board(本变更新增/修改的规则)

> 合入后并入 `openspec/specs/board/spec.md`。

## 接口(修改)
- `GET /api/skills?days={7|30|90}` → 去掉 `0`(全部)档;响应**新增 `today`**(UTC 当日 ISO 日期串)。
  `daily`/`table`/`funnel` 字段与口径不变。
- `GET /api/skill/{name}` → 响应**新增 `today`**(UTC 当日)。`daily`(used/equipped 分列)口径不变。

## 修改规则(MUST)
- 原前端规则"柱状图按 UTC 日,前端取窗口内使用量前 8 的 skill 分色……时间窗筛选只作用于柱状图"
  中的横轴部分**改为**:柱状图横轴按**所选时间窗逐日铺满**——右端 = 服务端 `today`,左端 =
  `today-(N-1)`,N ∈ {7,30,90};窗口内**每一天都占一个槽位**,有 used 数据才长堆叠柱,无数据留空槽。
  Top8 + 其它分色、按总量取峰值标定、轴标签稀疏化等沿用原状。
- 时间窗选择器**不再含「全部」档**,仅 7 / 30 / 90 天,默认 30。
- 铺满**只填真实的 0**,前端不得伪造序列;空白天如实留白(不压缩、不前移)。
- **今日柱(横轴最后一格)**用"进行中"视觉(斜纹 / 半透明 + 描边)区分于历史完整日,
  并在其明细浮窗中标注"今日进行中"。
- 横轴右端必须取服务端 `today`(UTC),不得用浏览器本地时间推算,避免跨 UTC 午夜错格。

## 新增规则(MUST)
- 悬停某日柱(整列)→ 该列正常、其余列降透明(整列高亮),并弹出**跟随光标的明细浮窗**:
  顶部当日日期,下列当天各 skill 按 sessions 降序(Top8 外并"其它"),分隔线后给**合计**
  (= 当天所有 skill 的 used sessions 之和,与柱总高同口径)。浮窗替代原生 `<title>` 提示,
  碰视口右 / 下边界时翻转。
- 移动端(无悬停)→ 点击某日柱显示浮窗,点击别处关闭。
- 单 skill 详情页趋势图同样按时间轴逐日铺满,**固定最近 30 个 UTC 日**(右端 = `today`);
  used 柱 + equipped 折线不堆叠相加;空天留白;今日柱进行中;悬停浮窗给出当天 used / equipped。
- **整窗全空**(选定窗口内无 used,或经搜索 / runtime / 来源筛选后无 used)→ 图区显示**空态文案**,
  不渲染一排空轴;只要窗口内 ≥1 天有数据,即按窗口逐日铺满,其余天留空槽。
- 筛选条(搜索 / runtime / 来源)作用后,某天归零则该天为**空槽**;图与主表口径一致联动。

## 可验证行为(新增)
- 只有今天有 used → `days=7` 主图渲染 7 个日槽,仅最后一槽有柱且标"进行中",其余留白;切 `days=30` → 30 槽同理。
- 数据有空档(今天 / 前 2 天 / 前 5 天有量)→ 空档日保留空槽,不压缩、不前移。
- 悬停任一日柱 → 浮窗列出当天各 skill(降序,Top8 外并"其它")与合计;合计 = 当天 used 会话总数。
- 切换 7 / 30 / 90 → 横轴日槽数随之变为 7 / 30 / 90;`table` 与 `funnel` 不受影响。
- 窗口选择器不含「全部」档。
- `/api/skills` 与 `/api/skill/{name}` 响应均含 `today`(= UTC 当日);其余字段与本变更前一致。
- 把当前时间置于 UTC 次日凌晨(或 mock `today`)→ 今日柱仍落在横轴最后一格,不错位。
- 进入任一 skill 详情 → 趋势图铺满最近 30 个 UTC 日,首见日之前的槽留白,今日柱进行中。
- 选定窗口内零 used(或经筛选后零)→ 图区显示空态文案,**不显示空轴**。
- 加 runtime / 来源 / 搜索筛选致某天归零 → 该天变空槽,其余有数据天照常渲染。
- 看板与 Agents 视图不受影响,原有 2 秒轮询行为不变。
