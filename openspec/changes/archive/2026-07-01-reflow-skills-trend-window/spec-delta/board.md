# spec delta:board

> 合入后更新 `openspec/specs/board/spec.md`。

## 修改接口/参数规则
- `/skills` 前端默认窗口从 `30d` 改为 `7d`。URL 无 `w` 且无合法旧 `win` 参数时，前端按 `w=7d` 请求和展示。
- 旧参数 `win` 继续兼容: `win=7/30/90` 分别映射为 `7d/30d/90d`；非法或缺失时 fallback 为 `7d`。
- `custom` 窗口参数无效时 fallback 为 `7d`。

## 修改前端规则(MUST)
- SKILLS 总览信息架构改为:
  控制条 -> KPI 环带 -> 治理健康条 -> 主分析区(排行/操作员排行 + 每日使用趋势图 | 治理待办) ->
  归因 Donut -> 明细表+抽屉 -> 公司库采纳漏斗。
- 「每日使用趋势图」不再全宽独占一层；它必须位于「使用排行 / 操作员排行」下方，作为排行的时间解释图。
- 桌面 `>1080px` 下主分析区为两列:左列包含排行和趋势图，右列为治理待办。
- 平板 `601px-1080px` 与手机 `<=600px` 下主分析区单列，顺序为排行 -> 趋势图 -> 治理待办。
- 趋势图的日期轨道长度与图表视窗尺寸必须解耦:
  - 单日槽宽固定，按当前 30d 观感定标。
  - `today`、`this_week`、`last_week`、`7d` 或任意 1-7 天窗口右对齐显示，不拉伸到整行。
  - `30d`、`90d`、`custom>7d` 可在图表内部横向滚动，默认滚到最右侧最新日期。
  - 页面根节点不得因趋势图产生横向滚动；横向滚动只能发生在图表容器内部。
- Skill 视角下，排行选中态继续联动趋势图选中 segment。
- Operator 视角下，操作员排行行继续点击进入 `/operator/:name`，不得新增与行跳转冲突的选中态；趋势图展示当前筛选后的 operator 分布。
- 筛选条(搜索/runtime/来源/Top N/隐藏 0 使用)变化后，排行和趋势图必须使用同一过滤口径；筛选后全空继续显示空态。

## 可验证行为(新增/修改)
- 打开 `/skills` 且 URL 无窗口参数 -> 时间窗口为 `7d`。
- 打开 `/skills?win=30` -> 时间窗口为 `30d`；打开 `/skills?w=90d` -> 时间窗口为 `90d`。
- `7d` 下趋势图位于排行下方，7 个日槽保持 30d 单日槽宽并右对齐，不被拉伸到整行。
- `30d` / `90d` 下趋势图默认显示最新日期；向左滚动可查看更早日期。
- 1440px 宽桌面下:左列为排行+趋势，右列为治理待办。
- 768px 平板下:排行、趋势、治理待办上下排列，页面根无横向滚动。
- 375px 手机下:排行摘要行之后是趋势图；`30d/90d` 仅图表内部横滚，页面根无横向滚动。
- Skill 视角点击排行某 skill -> 趋势图中该 skill 高亮，其他 segment 降透明；再次点击取消。
- Operator 视角点击操作员行 -> 进入 `/operator/:name`，不被趋势图选中态拦截。
