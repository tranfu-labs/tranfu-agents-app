# 设计:skills-responsive-layout

## 方案
本变更只做 SKILLS 统计域前端响应式适配,不改变数据获取、URL query、聚合口径或下钻路由。

### 覆盖范围
- `/skills`:总览页的视角卡、筛选条、趋势图、排行表、治理 Lens、公司库漏斗。
- `/skill/:name`:单 Skill 详情的指标卡、趋势图、runtime/operator 分布、最近记录。
- `/operator/:name`:单操作员详情的指标卡、趋势图、runtime 分布、使用 Skill 排行、最近记录。

### 断点规则
- 桌面 `>1080px`:保持现有布局,只修补可能导致溢出的 `min-width` / `max-width`。
- 平板 `601px-1080px`:
  - `/skills` 的排行和漏斗从左右分栏改为单列,排行在上、漏斗在下。
  - 筛选条可保持多控件同行,但允许换行,每个控件不小于可读宽度。
  - 指标卡使用 3-4 列,具体按页面字段数自动压缩。
  - 分布区使用单列。
- 手机 `≤600px`:
  - 筛选条单列,输入与下拉宽度 100%。
  - 视角卡说明文案保留可见,分段按钮等分满宽。
  - 趋势图在 `.chart-box` 内部横向滚动;禁止 `body` 横向滚动。`/skills` 的 7 天窗口应优先铺满手机宽度,
    30/90 天窗口允许内部横滚;详情页固定 30 天,允许内部横滚。
  - 可下钻排行表改为摘要行,首行展示名称 + 来源/主状态,次行展示 7d/30d/总计/用户/最近等关键指标。
  - 最近记录表改为摘要行,首行时间,次行主体字段,末行 runtime/mode/session。
  - 指标卡 2 列。

### 实现要点
- `styles.css`
  - 新增平板断点 `@media (max-width:1080px) and (min-width:601px)` 或扩展现有 ≤1080 规则。
  - 将 SKILLS 统计域加局部容器规则,避免影响全站普通表格。
  - 移动端表格用 CSS 摘要形态:保留 `<table>` 语义,在窄屏下隐藏 `thead`,让 `tr` 呈现为块状摘要行;单元格通过 `data-label` 或列选择器展示关键字段。
  - 保留 `records-table` 不可点击语义,避免手机摘要行误导。
- `Skills.tsx`
  - 给总览排行表、治理未收录表、操作员排行表的关键 `td` 增加用于移动端展示的类名或 `data-label`。
  - 治理 Lens 在手机下按钮换行,说明文案独占一行。
  - 总览页 `.split` 在平板和手机保持单列,排行在漏斗前。
- `SkillDetail.tsx`
  - 给最近记录单元格增加移动端标签/类名。
  - 指标卡只通过 CSS 调整列数,不改数据。
- `OperatorDetail.tsx`
  - 给使用 Skill 排行表和最近记录表增加移动端标签/类名。
  - 继续沿用现有 `dist-mirror` 在 ≤1080px 单列的规则。
- `Charts.tsx`
  - 保持当前 SVG 数据模型;按窗口调整最小宽度策略:`/skills` 7 天窗口优先适配容器宽度,
    30/90 天与详情 30 天保留内部横滚。继续确保浮窗视口避让在平板/手机可用。

### 测试门槛判断
- 本变更不改可测业务逻辑,主要是标记结构与 CSS 响应式规则。
- 若实现中单文件 CSS diff 超过 200 行,仍归为纯展示/CSS 变更,不强制单元测试;必须完成 AI/人工视口验证。
- 如实现中新增了数据转换函数或跨文件状态逻辑,再补充对应单元测试。

## AI 验证流程
- `npm --prefix frontend run build`
- 启动前端或由 FastAPI 服务本地预览后,检查以下视口:
  - 手机:375x812
  - 平板:768x1024
  - 桌面:1440x900
- 覆盖路径:
  - `/skills?view=skill&lens=all`
  - `/skills?view=skill&lens=untracked`
  - `/skills?view=operator`
  - `/skill/<已有 skill 名>`
  - `/operator/<已有 operator 名>`
- 每个视口检查:
  - `document.documentElement.scrollWidth <= window.innerWidth + 1`
  - 图表横向滚动只发生在 `.chart-box` 内。
  - `/skills?win=7` 在手机下趋势图无需整页横滚且应尽量铺满容器;30/90 天可在 `.chart-box` 内横滚。
  - 手机排行/最近记录为摘要行,点击可下钻排行行仍可用,最近记录仍不可点。
  - 暗/亮主题下文字不重叠、不截断关键数字。
  - 保存手机/平板/桌面关键页面截图用于人工复核。

## 权衡
- **保留 table 语义,用 CSS 做手机摘要形态**:可避免重写表格为两套 DOM,也保留桌面排序/键盘可达行为;代价是 CSS 会更细,需要给关键单元格补标签或类名。
- **平板不强制卡片化表格**:平板宽度足够承载表格 + 内部横滚,过早卡片化会降低扫描效率。
- **不处理其它页面**:本次只收敛 SKILLS 统计域,避免把移动端适配扩成全站大改。

## 风险
- 移动端表格 CSS 可能影响非 SKILLS 表格 → 使用 SKILLS 页面局部 class 限定。
- 图表浮窗在手机横滚场景中定位不准 → 验证滚动时关闭浮窗,点击柱子重新打开。
- 长 skill/operator 名称仍可能撑宽 → 对摘要行主标题使用 `overflow-wrap:anywhere` 和 `min-width:0`。
- 回滚方式:纯前端和文档变更,可 revert 本 change 对应提交。
