# spec delta:board

> 合入后更新 `openspec/specs/board/spec.md`。

## 修改前端规则(MUST)
- `/skills` 主分析区不再固定为「左:排行+每日使用 / 右:待处理线索」。
- `/skills` 必须按当前时间窗口长度切换主分析区布局:
  - `today`、`this_week`、`last_week`、`7d`、`14d`、有效 `custom<=14天` 等短窗口:
    桌面 `>1080px` 下,「排行 Bar / 操作员排行」与「每日使用趋势图」在主分析区整宽内左右并列。
  - `30d`、`90d`、有效 `custom>14天` 等长窗口:
    桌面 `>1080px` 下,「排行 Bar / 操作员排行」与「每日使用趋势图」在主分析区整宽内上下堆叠。
- 「待处理线索」必须从主分析区侧栏拆出为独立治理行。Skill 视角下固定三类:
  1. 有使用但未收录
  2. 装了 W 内没用
  3. 收录但零装机
  每一类必须呈现为独立区块,保留计数、Top items、证据/找人/忽略/查看全部等非破坏动作。
- 按人视角下的待处理/治理信号也必须使用独立治理行,不得重新挤占排行或趋势图宽度。
- 短窗口趋势图必须填满自身面板可视宽度,不得只以固定 30d 单日槽宽渲染成右侧窄条;同时柱体宽度必须有上限,避免 `today` 单柱过粗。
- 长窗口趋势图继续使用固定单日槽宽,仅在 `.chart-box` 内部横向滚动,并默认显示最新日期。
- 平板 `601px-1080px` 下 `/skills` 主内容必须单列:
  控制条 -> 过去 W 变化 -> 问题线索 -> 排行/操作员排行 -> 每日使用趋势图 -> 待处理线索 -> 归因 -> 明细 -> 公司库漏斗。
- 手机 `<=600px` 下必须保留首屏判断流:
  控制摘要 -> 问题线索 -> 待处理线索 -> 排行/操作员排行 -> 每日使用趋势图 -> 过去 W 变化 -> 归因 -> 明细 -> 公司库漏斗。
- 页面根节点不得因为排行、趋势图或待处理线索产生横向滚动;长窗口横向滚动只能发生在趋势图 `.chart-box` 内。
- Skill 视角下排行选中态继续联动趋势图 selected segment;Operator 视角下排行行继续下钻 `/operator/:name`,不得新增与行跳转冲突的选中态。

## 可验证行为(新增/修改)
- 1440x900 打开 `/skills?w=7d&view=skill`:主分析区中左侧为排行 Bar,右侧为每日使用;待处理线索位于其下一行,三类分别为独立区块。
- 1280x800 打开 `/skills?w=14d&view=skill`:仍为排行/每日使用左右布局;每日使用填满右侧面板,不出现大面积空白。
- 1440x900 打开 `/skills?w=30d&view=skill`:排行 Bar 独占一行,每日使用在其下方独占一行;待处理线索再下一行独立呈现。
- 1440x900 打开 `/skills?w=90d&view=skill`:每日使用全宽区域内部横滚,默认显示最新日期,页面根无横向滚动。
- 768x1024 打开 `/skills?w=7d`:主内容单列,顺序为排行 -> 每日使用 -> 待处理线索;待处理线索分类为上下区块。
- 375x812 打开 `/skills?w=7d`:页面根无横向滚动,第一屏能看到问题线索和待处理线索;排行摘要行之后显示 7 天趋势图,
  趋势图填满自身面板可视宽度但柱体不过粗。
- 375x812 打开 `/skills?w=30d`:页面根无横向滚动,30d 趋势图仅在 `.chart-box` 内部横滚。
- `/skills?view=operator&w=7d`:操作员排行与每日使用按短窗口规则布局;点击操作员排行行进入 `/operator/:name`;待处理线索独立行不拦截行跳转。
- 搜索/runtime/来源/Top N/隐藏 0 使用筛选变化后,排行与每日使用继续使用同一过滤口径,待处理线索继承当前窗口和可用筛选语义。
