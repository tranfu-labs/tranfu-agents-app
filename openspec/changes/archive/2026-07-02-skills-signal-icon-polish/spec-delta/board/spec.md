# spec delta：board

## 修改规则(MODIFIED)
- 按 Skill 视角「问题线索」5 项仍为未收录占比、装了没用比例、公司库覆盖率、Top3 集中度、平均 skill/会。每项展示当前事实值和 icon 证据入口，**不得再显示可见动作文案**，例如「看未收录 used 名单」「看已装未用名单」「看集中使用分布」。具体 records/items/names 仍只能出现在待处理线索、证据页或详情抽屉中。
- 按人视角「使用线索」同样只展示当前事实值和 icon 证据入口，**不得显示可见动作文案**，例如「看操作员名单」「看 runtime 分布」。
- 所有 evidence 跳转 icon 默认态应为浅灰弱提示色；悬浮和键盘 `focus-visible` 时恢复当前高亮色，并提供可见交互反馈。icon 的 `aria-label` / tooltip 必须继续表达「查看证据」语义。

## 可验证行为(新增)
- 1440x900 打开 `/skills?view=skill&w=7d` → 「问题线索」区域不出现「看未收录 used 名单」「看已装未用名单」「看有证据的公司库 skill」「看集中使用分布」「看会话使用分布」等动作文案；每项仍有一个 evidence icon。
- 1440x900 打开 `/skills?view=operator&w=7d` → 「使用线索」区域不出现「看操作员名单」「看会话使用分布」「看集中使用分布」「看 runtime 分布」等动作文案；每项仍有一个 evidence icon。
- 鼠标悬浮或键盘聚焦任一 evidence icon → icon 从默认浅灰切换为当前高亮色，并保持原跳转目标。
- 375px 手机宽度下，问题线索/使用线索不因旧动作文案产生横向滚动；页面根无横向溢出。
