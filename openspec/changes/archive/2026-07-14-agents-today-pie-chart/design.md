# 设计：agents-today-pie-chart

本文件描述实现与权衡。视觉三段文档位于 `visual/agents/`，字符线框见 `wireframes.md`，事实增量见 `spec-delta/board/spec.md`。

## 方案

### 1. 单日模式判定

沿用 `resolveAgentChartMode`：当前指标全窗为零时返回空态；只有一个真实统计日且指标为正值时进入单日模式；两个及以上统计日进入序列模式。单日模式只改变图形，不改变趋势模型与统计口径。

### 2. 扇区模型

新增纯函数读取单日 `segments`，按当前指标取正值分组，计算：

- 分组原值；
- 相对当前指标总值的比例；
- 在完整圆环上的累计起点。

函数保留现有 Top 8 + 其他的顺序与名称，不重新排序，也不把非当前指标混入扇区面积。浮层仍读取完整日模型，因此继续同时展示当日活跃 Agent 总数和活跃总时长。

### 3. 图表渲染与交互

- 单日模式渲染同心圆环轨道与多个分组弧段，弧段使用现有 `colorOf(segment.name)`，透明度与图例 hover/focus 联动。
- 圆环中心显示当前指标总值和指标名称；活跃时长使用现有 `dur` 格式化。
- 圆环保留一个透明日期命中环作为键盘停靠点，保持“一个日期只有一个 Tab 停靠点”的约束。弧段鼠标悬停可同步高亮对应图例并打开同一 tooltip；点击、聚焦、退出键、移出、滚动的关闭逻辑沿用现有图表。
- `agentChartAnchor` 从仅接收矩形泛化为接收任意 SVG 图形元素，以同一个视口避让算法服务柱子与圆环。
- 多日分支保留现有坐标轴、堆叠柱、日期抽样、今日斜纹与内部横滚，不改几何计算。

### 4. 响应式与主题

圆环的颜色、轨道、中心文字、图例、tooltip 全部使用现有 CSS 变量。桌面与平板保持趋势面板现有高度；手机缩小圆环尺寸并保持图例换行，页面根不产生横向滚动。深色、浅色与跟随系统主题共用同一变量体系。

## 测试

### 单元测试

- 单日扇区只包含当前指标为正值的分组。
- 扇区比例之和为 1，累计起点连续，原值之和等于单日总值。
- 切换活跃 Agent 数与活跃时长时使用各自字段，不复用错误比例。
- 单日、空态与多日模式判定保持原有边界。

### 浏览器验证

- `/agents` 默认今天：操作员视角与运行终端视角均显示环形扇形图。
- 切换活跃 Agent 数与活跃时长：扇区、中心值、tooltip 同步更新。
- 悬停扇区和图例：当前分组保持高权重，其他分组降权；tooltip 不溢出视口。
- 键盘 Tab 只有一个日期停靠点，聚焦显示 tooltip，退出键关闭。
- `w=7d` 与 `w=30d`：仍显示现有柱状图，短窗无根横滚，长窗只在图表盒内滚动。
- 深色、浅色与 375 像素手机视口均无布局破坏。

## 权衡

- 选择环形扇形图而非实心饼图：环形能在中心保留当前指标总值，视觉重量也更接近现有细柱和轻量描边。
- 不为单日构造小时序列：当前数据只有日级事实，伪造小时分布会误导用户。
- 不把单日扇形逻辑推广到 Skills 图表：本次需求只针对 Agents，避免改变 Skills 的既定每日使用语义。

## 风险

- 极小扇区可能难以直接命中；图例仍是稳定的鼠标与键盘高亮入口，整圈日期命中环保留 tooltip 入口。
- 分组数较多时图例可能换行；沿用现有 Top 8 + 其他上限并在手机端收紧间距。
- 浮层锚点从矩形泛化后可能影响多日柱；保留同一边界计算并通过多日回归验证。
