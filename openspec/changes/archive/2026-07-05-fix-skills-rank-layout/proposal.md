# 提案：fix-skills-rank-layout

## 背景
`/skills` 主分析区在短窗口桌面布局下存在两个全局布局缺陷：使用排行首行的长 skill 名会被单行截断，且使用排行与每日使用两张同排卡片的外框底边在桌面截图中错位。窄屏下长 skill 名还需要避免与数值、动作和条形轨道重叠。

## 提案
- 调整 `/skills` 主分析区排行行布局，让 `openspec-driven-development` 在 `1440x900` 桌面、`/skills?w=7d` 默认可见状态完整可读；可使用换行、软断行或更合理的行布局，不能只依赖 hover/title。
- 调整短窗口桌面主分析区网格，让使用排行与每日使用卡片同排时外框底边对齐；`1440x900` 下两者 `getBoundingClientRect().bottom` 差值必须不超过 `4px`。
- 加固 `375px` 窄屏排行行，避免长 skill 名、数值、来源/记录动作与条形轨道重叠或撑出页面根横滚；若极端长名发生截断，必须保留 title、aria-label 或行详情等完整可读路径。

## 影响
- 影响模块：M2 看板前端 `frontend/`，主要是 `/skills` 主分析区展示层样式。
- 不改变 `/api/skills` 统计口径、排序、筛选、请求参数、持久化状态或服务端协议。
- 不重排整个 SKILLS dashboard；仅修复排行行可读性、同排卡片底边对齐与窄屏抗溢出。
