# 设计:skills-operator-view

> 在 skills-stats-page 已落地的 SKILLS 总览/详情之上,增量加"按人视角"。
> skill 视角的口径与线框见 skills-stats-page/design.md,本文件只描述增量。

## 锁定口径(2026-06-16 访谈逐项拍板,实现时不得擅改)
- **整页切主语**:总览页顶部加 `[ 按 skill ] / [ 按人 ]` 视角切换;切换后柱状图、主表、下钻
  **整页同一主语**,不混搭(不会出现"上面按人、下面按 skill")。唯一不切的是筛选条,仅搜索框
  提示语随视角变(搜索 skill 名 ↔ 搜索操作员)。
- **人维度计量** = 会话×skill 去重使用次数(沿用 `skill_uses` 主键 `(session,skill,mode)`),
  语义为"此人在多少个会话里用过 skill",**非真实调用次数**;界面须标注此口径。
- **只 used**:人视角彻底不碰 equipped(装备态只在单 skill 下钻出现);仅在 OpenClaw 装备过、
  从未 used 的人不进人榜。
- **operator 空值排除**:采不到操作员的会话不进人榜、不进按人柱状图(不做"未识别"兜底,避免
  一个大灰段盖过真人)。
- **时间窗统一默认 30 天**:两个视角同默认,切视角**不重置**时间窗;人主表与 skill 主表完全
  对称——同样固定 7天/30天/累计三列,时间窗只控柱状图,默认按 30 天降序。
- **柱状图随视角换分段维度**:skill 视角按 (日×skill) 分段、人视角按 (日×operator) 分段;
  Top8+其它合并、配色、悬浮高亮逻辑原样复用,仅分段 key 不同。
- **漏斗常驻、不随视角变**:漏斗是"公司库健康"独立面板(catalog→安装→使用),与当前视角主语
  解耦,两个视角都照常显示同一套 skill 漏斗。
- **双向下钻闭环**:skill 下钻看"谁用了它"、人下钻看"他用了啥";人详情的 skill 排行行可点回
  skill 下钻(人→skill→人 互相穿透)。

## 改动文件与职责

### `server/app.py`
1. **`GET /api/skills` 人维度聚合**(供按人视角,均只 used、排除空 operator):
   - `operator_table[]`:每 operator 一行——7天/30天/累计使用次数(会话×skill 去重)、用过 skill
     去重数、会话数、各 runtime 计数、近 14 天逐日序列(趋势小图)、最近使用日。默认按 30 天降序。
   - `operator_daily[]`:逐 (日 × operator) 的 used 计数,供按人柱状图;Top8+其它合并在前端做。
   - `days` 参数沿用现有语义,仅影响 daily 类字段(含 `operator_daily`),不影响 table 类。
2. **`GET /api/operator/{name}`**(单人详情):指标(7/30/累计使用次数/用过 skill/会话/首见/最近)、
   日级序列(按 skill 分段)、该人 skill 排行(只 used,含来源映射)、runtime 分布、
   最近记录(默认 50 条:日期/skill/runtime/session_id,模式恒 used)。查无此人 → 404。
3. skill 视角既有两个端点行为不变;`/api/skills` 仅新增字段,向后兼容。

### `frontend/`
- 总览视图加"视角切换"状态(按 skill / 按人):两视角共用筛选条与堆叠柱状图组件,柱状图分段
  key 参数化(skill ↔ operator);按人时主表换成人排行、漏斗照常渲染。时间窗两视角统一默认 30 天,
  切视角不重置。搜索框提示语随视角变。
- operator-detail 视图:拉 `/api/operator/{name}`;日趋势按 skill 分段;⑨ 左右两栏(skill 排行
  + runtime 分布,与 skill-detail 同布局);页头两行(返回 + 标题);skill 排行行点击跳
  `/skill/:name` 实现双向下钻;返回回按人总览。路由扩展 `/operator/:name`。
- 暗/亮双主题与 ≤600px 窄屏遵循现有样式约定。

## 性能与降级
- 聚合读时现算(与 skill 视角同风格);按 (日×operator)/(operator) 分组,量级与 skill 维度同级,足够快。
- operator-detail 与 `operator_*` 字段不进 2 秒主轮询;按人视角进入时加载、低频刷新,失败显示错误态。

## 依赖与顺序
- 前置:skills-stats-page 已上线(总览/详情骨架与柱状图组件已存在)。`skill_uses.operator` 已采集,
  数据已就绪,无采集侧前置变更。
- 实施顺序:`/api/skills` 加 `operator_*` 字段 → 总览视角切换 + 人主表 → `/api/operator/{name}`
  + operator-detail 视图 → 双向下钻联调。
